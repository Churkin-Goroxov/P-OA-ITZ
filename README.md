## 1. Название проекта
### Автоматизированное распределение учебной нагрузки преподавателей

Проект разработан в рамках дисциплины **Программирование и Основа Алгоритмизации** и предназначен для автоматизации 
переноса распределения учебной нагрузки между преподавателями на основе данных предыдущего учебного года.

Программа анализирует сведения о нагрузке прошлого года, сопоставляет их с нагрузкой текущего года и автоматически 
выполняет распределение часов между преподавателями в соответствии с условиями задания.

________________________________________________

## 2. Цель работы

**Разработать программное средство для автоматизированной обработки данных учебной нагрузки и реализации алгоритма 
распределения часов между преподавателями на основе статистики предыдущего года.**

**В ходе выполнения работы необходимо:**

- *организовать чтение данных из Excel-файлов;*
- *представить данные в виде объектной модели;*
- *реализовать поиск совпадающих нагрузок между учебными годами;*
- *выполнить агрегацию одинаковых записей нагрузки;*
- *реализовать алгоритм пропорционального распределения часов;*
- *сформировать итоговый Excel-файл с результатами обработки.*

Проект должен работать без ручного вмешательства и корректно обрабатывать произвольное количество записей в исходных 
файлах.

____________________________________________

## 3. Постановка задачи

В распоряжении пользователя находятся данные о распределении учебной нагрузки за предыдущий учебный год, а также данные
о нагрузке текущего учебного года.

Необходимо автоматически определить, ***каким преподавателям должна быть назначена нагрузка текущего года на основании 
распределения, существовавшего ранее.***

**В рамках варианта задания реализуется следующий алгоритм:**

1. *Для каждой записи текущего года выполняется поиск аналогичной нагрузки в данных прошлого года.*
2. *Определяется список преподавателей, участвовавших в выполнении данной нагрузки ранее.*
3. *Вычисляется суммарная нагрузка каждого преподавателя.*
4. *Если в распределении участвовало более двух преподавателей, выполняется пропорциональное распределение нагрузки 
текущего года между ними.**
5. *Результат сохраняется в отдельный Excel-файл.*

Дополнительно из обработки исключаются специальные виды нагрузки, требующие отдельного подхода к распределению.

## 8. Алгоритм работы программы

### Общая схема работы

Программа выполняет обработку учебной нагрузки в несколько последовательных этапов.

__________________________________________

### Этап 1. Чтение исходных данных

**С помощью класса Reader загружаются три Excel-файла:**

- ***нагрузка прошлого учебного года;***
- ***нагрузка осеннего семестра текущего года;***
- ***нагрузка весеннего семестра текущего года.***

Каждая строка таблицы преобразуется в объект `LoadRecord.`

_____________________________________________

### Этап 2. Формирование нагрузки текущего года

***Списки записей осеннего и весеннего семестров объединяются в один список.***

***В результате формируется полный набор нагрузок текущего учебного года.***

____________________________________________

### Этап 3. Агрегация данных

***Модуль `LoadAggregator` объединяет одинаковые записи нагрузки.***

***Совпадающими считаются записи с одинаковыми:***
- *группой;*
- *дисциплиной;*
- *видом нагрузки.*

Количество часов таких записей суммируется.

**Цель этапа** — исключить дублирование нагрузки и подготовить данные к дальнейшей обработке.

____________________________________________________

### Этап 4. Исключение специальных нагрузок

####Перед выполнением распределения программа исключает нагрузки, которые не относятся к условиям задания.

**К таким нагрузкам относятся:**
- дипломные работы;
- ВКР;
- магистерские работы;
- руководство кафедрой.

Данные записи не включаются в итоговый результат.

_________________________________________________

### Этап 5. Поиск совпадений

#### Для каждой записи текущего года выполняется поиск соответствующих записей прошлого года.

***Поиск осуществляется по ключу:***
`(group, discipline, load_type)`

Для ускорения поиска используется предварительно построенный индекс.

_________________________________________

### Этап 6. Подготовка данных для распределения

**Из найденных записей удаляются:**
- записи без преподавателя;
- записи с неположительным количеством часов.

**После фильтрации определяется:**
- список преподавателей;
- суммарная нагрузка каждого преподавателя;
- общая нагрузка по данной дисциплине.

_______________________________

### Этап 7. Проверка условия задания

####Распределение выполняется только в том случае, если в прошлом году данную нагрузку выполняли более двух преподавателей.

Если преподавателей два или меньше, нагрузка в итоговый результат не включается.

______________________________

### Этап 8. Пропорциональное распределение часов

**Для каждого преподавателя вычисляется доля участия:**

`доля = часы преподавателя / общая нагрузка`

**После этого определяется количество часов текущего года:**

`новые часы = нагрузка текущего года × доля`

**На основе вычисленных значений создаются новые записи нагрузки.**

## 9. Проектные ограничения

###1. Остановка чтения на первой полностью пустой строке

**Чтение Excel-файлов выполняется до первой полностью пустой строки таблицы.**

Полностью пустой считается строка, в которой отсутствуют значения во всех основных столбцах:

| Группы       | 
|--------------|
| Дисциплина   |
| Вид нагрузки |
| Нагрузка     |

**После обнаружения такой строки чтение файла прекращается.**

> *Данное решение принято осознанно, поскольку после разрыва невозможно однозначно определить, является ли следующий 
блок продолжением таблицы или новой независимой таблицей.*

### 2. Исключение итоговых строк

**Строки, содержащие только итоговые значения часов и не имеющие информации о дисциплине, группе или виде нагрузки, 
не рассматриваются как учебная нагрузка и исключаются из обработки.**

### 3. Ограничение варианта задания

**В итоговый результат включаются только нагрузки, удовлетворяющие условию задания:**

- количество преподавателей в прошлом году больше двух.
- тид не является специальным видом нагрузки:
  - диплом;
  - ВКР; 
  - магистр; 
  - магистрант; 
  - руководство.

**Все остальные случаи исключаются из итоговой таблицы.**

**Такие нагрузки требуют отдельного подхода к распределению и не входят в реализованный вариант задания.**

## 11. Запуск проекта

### Пути к файлам задаются в модуле main.py.

#### Изменение входных файлов

**При использовании других файлов необходимо изменить пути в следующих строках:**

```python
old_records = reader.read("src/data/15 вариант.xls")

autumn_records = reader.read("src/data/Список 15вар - осень.xls")

spring_records = reader.read("src/data/Список 15вар - весна.xls")
```
